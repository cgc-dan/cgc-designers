CGC - Editor Compatibility

CGC Editor is an online tool provided by Camgirl.Cloud

It allows editing pre-designed templates without writing a single line of code

Because it is not a website builder but a template editor, the tool has options and features which a registered member can use

Some capabilities of this tool are limited by it's current development or denied for security reasons.

OS, Browser and Device compatibility

This are our suggestions regarding Editor’s compatibility with OS, browsers, devices...

For best performance and usability we strongly recommend:

  • Operating system – MAC, Windows
  • Browser – Chrome (best performance – recommended), Firefox (best performance), Opera, also tested with Safari (with Safari browser known to have some issues – read more about this issue on our forum)
  • Devices – Desktop (laptop or pc for best performance). Feel free to test the Editor with mobile devices too but at this moment the editing process does not support touch functionality at best performance and the editor will become difficult to use.
  • Proxy – please avoid using proxy services (browser extensions, self-installed app, hide ip or online services). Using proxy services you might not be allowed to use some functions.
  • Browser extensions – You can have enabled extensions (modules) on your browser but we highly recommend disabling them at least when you are using our service.

AdBloker for example, might detect some images as banners/ads , in this case you won't be able to edit.
At this moment our service does not run ads (pop-ups, pop-under) on the site so its safe for you to work.

Template compatibility

MyFreeCams

MyFreeCams templates added to Editor for live editing have two components, the HTML markup and CSS additional markup

In editing mode, the editor tool generates both markup languages separately. Upon installing a MyFreeCams profile design, both markup codes must be added

Wrapper / parent - all HTML markup code must be included in a div parent with "main" as id


<div id="main">...content/full HTML markup...</div>

Chaturbate

As mentioned in Chaturbate documentation, upon submitting a template, we've added the Editor Margin which is a fix for better positioning of the submitted design inside Editor tool (do not use this fix box option with MyFreeCams templates

Why it was needed? Because for better position of the layout installed on Chaturbate platform, the main wrapper must have this margin values
margin:120px auto 50px -146px

to correct this and display the template inside the Editor a fix margin must be added
margin:-110px auto 50px 155px

Wrapper / parent - all HTML markup code must be included in a ul parent with "main" as id
<ul id="main" style="...">...content/full HTML markup...</ul>

Niteflirt/Camsoda

Niteflirt and Camsoda designs have a limited number of accepted characters which can be added by profile/bio owner inside Bio box - read more on forum topic

CAM4

CAM4 platform is currently very restrictive regarding markup editing which can be added to bio page, reason for which we've decided to offer only plain image designs

In this context, our editor is using editable HTML as Canvas with image output. When saving a CAM4 design, the Editor is generating a .png image file
