Quick intro

MyFreeCams is an adult live video-steaming platform.

Registered webcam performers and members of this website have the option to edit their own profile page design using images, HTML/CSS markup language

Accepted HTML tags

If you are (or not) familiar with MyFreeCams platform you already know or find out right now, that MFC has some HTML5 tag restrictions.

But most common HTML works like a charm

a – html tag used to define a link
<a href="

p – html paragraph tag
<p>

i – html tag used to render the text italic
<i>

strong – html tag used to render the text in bold style — similar to b html tag
<strong>

u – html tag is used to underline a defined text element
<u>

ul, ol, li – html tags used to render list styles and list types
<ul> / <ol> / <li>
<ul> <li>some text or link here</li> </ul>

h1, h2, h3... – html tags which defines heading text elements

img – html tag used to define and render image element
<img src="

div – html tag used to define a div element
<div

br – html element for brake – basically it starts a new line braking a paragraph, a text element or can add a space between other elements like graphics (images)
<br>

A very powerful feature implemented by MyFreeCams is the option of adding and using custom css.

Astonishing layouts can be created and installed on MyFreeCams enhancing the HTML with additional css
In this context user can make use of custom class to style-up their HTML elements

Check out MFC Cheat Sheat Stylesheet (EN)

background-image: url(image-path) – suggested
background:#f1f1f1 – not
background: url(image-path) – not

Suggestions

Avoid using fixed width, instead, when possible use % values in order to make the project as flexible (fluid width) as possible

Use and create class in css markup rather than using inline css

For best performance optimize images
